1. 产品概述

1.1 背景与痛点

在当代社会中，年轻人面临着日渐复杂的亲戚网与社交圈。由于平时聚少离多，用户在节假日面对庞大家族时，普遍存在以下痛点：

  • 关系健忘： 无法准确叫出远房亲戚、堂表兄弟姐妹及其下一代子女的名字或小名。
  • 关键信息模糊： 难以记住小辈的精准年龄以及“当前正就读几年级”，导致长辈在交流时缺乏共同话题。
  • 人情账目混乱： 无法高效追踪亲戚结婚、生娃、过年等随礼红包的往来详情，或在回礼时缺乏差额比对参考。
  • 情感记忆断层： 缺乏一个私密的个人空间，用来记录与特定重要亲友共同旅游、聚餐等升温经历。

1.2 核心产品定位

本产品是一款主打“纯本地、极致隐私、即用即走”的个人社交辅助管理工具。它既非宏大的宗族家谱，也非商务用的 CRM 系统，而是专为年轻人打造的“有温度的数字人脉手账”。

💡 商业化与变现策略 (香港个人开发者)
本应用不采取高门槛的强制订阅或买断制，而采用“免费下载 + 激励广告保底 + 自愿打赏去广告”的混合模式。鉴于开发者为香港居民，整体广告变现架构将无缝依托国际标准进行合规搭建。

2. 核心系统架构与商业变现

2.1 纯本地化技术边界与隐私设计

  • 数据存储： 所有业务数据（如成员姓名、出生日期、回忆文字、流水账目等）全部持久化存储于用户手机本地的 SQLite 数据库或加密 MMKV 键值对中。
  • 媒体资源： 动态回忆中添加的照片，应用内仅存储其在系统相册中的本地 URI 路径引用，不进行任何云端图片上传，从而实现 0 服务器运营成本与 100% 数据隐私安全。
  • 网络依赖： 应用除加载三方广告组件、版本更新提示或获取汇率外，核心业务逻辑在完全断网（飞行模式）状态下亦可流畅使用。

  3. 激励视频广告 (Rewarded Video Ad)： 用于高级衍生技能解锁（例如：一键导出 HTML 精美家谱报告）。用户点击后需完整观看 15~30 秒视频，即可免费解锁该功能 24 小时使用权。

3. 核心功能模块详细需求 (三个核心 Tab)

3.1 Tab 1：【家族】(层级树状与自动换算视图)

用于组织和查看具有血缘级联、辈分关系的家族成员。其核心产品逻辑在于对成员层级的直观可视化和对儿童成长状态的智能感知。

3.1.1 核心字段与底层逻辑

  • 每个成员节点需录入：真实姓名、小名/乳名、性别、出生年月日、父亲ID、母亲ID、配偶ID、自定义称呼。

3.1.3 杀手锏功能 2：称呼算法反向解析

传统称呼计算器为单向输入逻辑。本系统要求在【家族树】建立完毕后，当用户点开某个具体子节点（例如大表姐的女儿）的详情页时，系统自动根据级联关系，在下方小字部分进行反向渲染：
"你应该称呼她为：表外甥女" / "她应该称呼你为：表舅/表姨"。

3.2 Tab 2：【圈子】(平面网状朋友视图)

朋友、同事、客户等关系不属于血缘级联树，采用“扁平列表 + 聚类标签群”的方式组织。

  • 多标签聚合： 用户可为某位朋友打上多个复合标签，例如 #大学室友、#前公司同事、#骑行搭子。
  • 家属随行字段： 朋友卡片中提供扁平化的非树状随行备注，用于平移记录朋友家属，结构如：配偶: 王小五、一胎: 豆豆 (2022年10月生)。此功能可让用户迅速检索朋友的孩子，确保礼貌社交。

3.3 Tab 3：【回忆】(双向交互人情时光机)

此模块将原本冷冰冰的记账本升华为一个有温度的动态时间轴，全面支持金钱和非金钱社交事件的混合记录。

3.3.1 事件类型与定义

  1. 💰 物质往来 (金钱/人情账本)： 记录结婚随礼、生娃红包、新年压岁钱等。支持输入金额、选择收/支方向。
  2. 🚗 共同经历 (陪伴记录)： 记录诸如“和二表姐一家自驾去迪士尼”、“死党聚餐”等。
  3. 🎯 重要里程碑 (关注事件)： 记录如“外婆80大寿”、“大表姐儿子今日上小学”。

3.3.2 强耦合交互设计 (核心爽点)

  • 多联络人绑定： 任何一条【回忆】或【账目】记录，允许用户同时勾选多个人名 ID。例如：新建一条“两家自驾游”经历，勾选“大表姐”和“表侄子狗蛋”，此记录会同时、自动地分发渲染至这两个人的个人详情时光轴内。
  • 差额往来自动清算： 在某人的详情页中，系统会自动加总与该成员的所有金钱往来流水，并在顶部给出一个全局爽点统计面板："历史你支出 5,000 元，收到回礼 3,000 元。当前净人情往来：逆差 2,000 元"。

4. 纯本地数据导出、导入与备份方案

为向用户深度兑现“数据完全自主、无广告绑定危险”的承诺，系统不建立任何云数据库，全面提供全量备份与分类明细导出功能。

4.1 开发者底层备份：一键本地 WebDAV / JSON 打包

应用设置中提供“全量备份”按钮，一键将底层的 SQLite 核心关联表和配置信息，统一编码成一个名为 rel_backup_xxxx.json 的高密度数据文件。用户可将其自由保存至个人网盘或通过本地隔空投送（AirDrop）进行多设备间的数据迁移与导入恢复。

4.2 普通用户无门槛阅读：分类明细 CSV / Excel 导出规范

系统必须支持将三个 Tab 转换成标准的平铺式 CSV 文件，确保其能用 Excel 或 Numbers 完美打开查看。表格列定义必须严格依照下表设计执行：

表 1：家族树成员档案.csv (Tab 1 导出规范)

列名 (Column) 数据类型 列含义说明与生成逻辑
member_id INT (Primary Key) 本地成员唯一自增标识符，用于保持级联外键关系。
full_name VARCHAR 亲戚的官方/真实姓名（如：张美丽的女儿名称）。
nickname VARCHAR 家族常用的小名/乳名（如：狗蛋）。
relation_label VARCHAR 用户对该亲戚的自定口头称呼（如：大表姐、表外甥）。
father_id / mother_id INT (Nullable) 核心： 用于在表格或未来迁移中完美还原树状血缘关系。
spouse_id INT (Nullable) 关联配偶的 ID，用于标记姻亲关系（如表姐夫）。
birth_date DATE (YYYY-MM-DD) 出生年月日，以便电脑端或其他系统未来进行周岁二次换算。
memo_markdown TEXT Markdown 格式的文字手记、聊天避讳或个人喜好。

表 2：朋友与社交圈档案.csv (Tab 2 导出规范)

列名 (Column) 数据类型 列含义说明与生成逻辑
friend_id INT (Primary Key) 朋友节点的唯一独立标识。
friend_name VARCHAR 朋友的姓名或日常备注名。
tags_string VARCHAR 多标签聚合： 多个标签之间通过半角分号（;）拼装，如 "大学室友;驴友;数码圈"。
anniversary_date DATE (YYYY-MM-DD) 关键纪念日（如生日、相识纪念日）。
family_members_text TEXT 扁平家属信息串： 拼装输出其家属快照，如 "配偶:王五;儿子:大宝(2021-10)"。

表 3：社交动态与回忆档案.csv (Tab 3 导出规范)

列名 (Column) 数据类型 列含义说明与生成逻辑
event_id INT (Primary Key) 单条活动流水或人情往来的唯一标识。
bound_member_names VARCHAR 核心转换： 导出时代码自动查询并将关联的多个人名ID转换成明文文本，用分号隔开，如 "大表姐;表侄狗蛋"，方便 Excel 阅读。
event_type ENUM 固定输出：金钱往来 / 共同活动 / 关键里程碑。
event_title VARCHAR 事件摘要，例如 "大表姐结婚随礼" 或 "大屿山自驾环岛"。
amount_flow VARCHAR 资金流向，固定输出：收入 / 支出 / 无金钱交互。
amount_value DECIMAL (Nullable) 金钱往来的精准数额，活动类流水则显示为 0 或置空。
detail_notes TEXT 该笔回忆或活动的深度手记，例如 "这次聚餐表姐夫请吃了大餐，小辈很喜欢抓蜻蜓"。
photo_paths_local TEXT 本地照片的存储 URI 路径快照（如 "/storage/emulated/0/Android/data..."）。
occur_date DATE (YYYY-MM-DD) 该项事件发生的具体日期。
